Getting Started

Three steps to your first API call

Get up and running in minutes. Our TypeScript SDK provides full type safety, IDE autocompletion, and comprehensive documentation.

1

Register as a Developer

Create a Scholarly account and enable developer mode from the Developer Console. Choose your tier based on your usage needs.

2

Generate API Keys

Create API keys from the Developer Console. Each key is scoped to specific services and rate limits based on your tier.

3

Make Your First Request

Install the TypeScript SDK (npm install @scholarly/sdk) or call the REST API directly with your API key in the Authorization header.

// Install the SDK
npm install @scholarly/sdk

// Make your first API call
import { Scholarly } from '@scholarly/sdk';

const client = new Scholarly({
  apiKey: 'sk_dev_...',
});

const standards = await client.curriculum.search({
  query: 'fractions year 5',
  state: 'WA',
});

Requirements

Minimum requirements for developers

All apps and integrations must meet these technical and compliance standards before publishing to the Scholarly marketplace.

Technical Requirements

  • All API communication over HTTPS (TLS 1.2+)
  • OAuth 2.0 authentication with PKCE for public clients
  • Respect rate limits and implement exponential backoff
  • Handle API errors gracefully with user-friendly messages
  • Webhook endpoints must respond within 5 seconds
  • Support API versioning (current: v1)

Compliance Standards

  • COPPA compliance for apps accessed by children under 13
  • FERPA compliance for apps handling student education records
  • Australian Privacy Act 1988 compliance for Australian users
  • No collection of personal data beyond stated purposes
  • Data retention policies aligned with Scholarly standards
  • Mandatory breach notification within 72 hours

Quality Standards

  • App must have a clear educational purpose
  • WCAG 2.1 AA accessibility compliance
  • Responsive design supporting desktop and tablet
  • Error-free operation under normal usage conditions
  • Documentation including setup guide and API usage
  • Support contact and issue reporting mechanism

Content Guidelines

  • Age-appropriate content for the targeted year levels
  • No advertising or tracking within educational contexts
  • Australian English spelling and terminology preferred
  • Curriculum references must align with ACARA standards
  • AI-generated content must be clearly labelled
  • No misleading claims about educational outcomes
